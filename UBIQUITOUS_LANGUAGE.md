# Ubiquitous Language

## Core CLT Terms

| Term | Definition | Aliases to avoid |
| --- | --- | --- |
| **Intrinsic Load** | The working-memory demand caused by the essential interacting elements of the material relative to the learner's prior schemas. | Difficulty, hard content, content load, complexity |
| **Extraneous Load** | Avoidable working-memory demand caused by instructional design that does not help the learner build the target schema. | Bad writing, clutter, noise, unnecessary complexity |
| **Germane Processing** | Schema-relevant mental activity such as explaining steps, comparing examples, abstracting rules, or applying a pattern. | Germane load, productive load, good load |
| **Schema** | An organized knowledge structure in long-term memory that lets related elements be treated as one meaningful unit. | Mental model, concept map, memory chunk |
| **Worked Example** | A fully solved example that shows the goal, steps, and reasoning for a target task before independent practice. | Example, sample answer, demonstration |
| **Expertise Reversal** | The pattern where instructional support that helps novices becomes redundant or harmful after the learner has sufficient schemas. | Too much help, advanced learner effect, scaffolding fatigue |

## Relationships

- **Intrinsic Load** belongs to a learner-material pairing: the same material can impose lower **Intrinsic Load** for a learner who already has relevant **Schemas**.
- **Extraneous Load** belongs to the design of the material: reduce it before adding more practice, examples, or explanation.
- **Germane Processing** is promoted by design moves, but should not be treated as a directly measured load quantity.
- A **Worked Example** helps a novice build a **Schema** by reducing search and making the target structure visible.
- **Expertise Reversal** determines when to fade or remove supports such as **Worked Examples**, extra explanations, and integrated guidance.

## Example Dialogue

> **Designer:** "The learner is new to the topic, so the **Intrinsic Load** is high because too many elements interact at once."
>
> **Domain expert:** "Then reduce **Extraneous Load** first: clean the order, integrate labels with diagrams, and remove repeated text."
>
> **Designer:** "Should I start with practice problems?"
>
> **Domain expert:** "Use a **Worked Example** first so the learner can see the target **Schema** before solving alone."
>
> **Designer:** "When the learner can already explain the pattern, I should fade the worked steps because of **Expertise Reversal**."

## Flagged Ambiguities

- "Difficulty" is ambiguous: use **Intrinsic Load** for essential element interactivity, **Extraneous Load** for avoidable design friction, or missing **Schema** when the learner lacks prerequisites.
- "Germane load" suggests a separate measurable load; prefer **Germane Processing** and describe the design move that promotes schema-relevant activity.
- "Example" is too broad; use **Worked Example** only when the solution is complete enough to show steps and reasoning before independent practice.
- "Expert" means the learner has the relevant **Schema** for this target material, not that the learner is generally advanced or professionally qualified.
